Q. Difference  in echo and print?
A.	Speed - There is a difference between the two, but speed-wise it should be irrelevant which one you use. 
   echo is marginally faster since it doesn't set a return value if you really want to get down to the nitty gritty.
	
   Expression - print() behaves like a function in that you can do: $ret = print "Hello World"; And $ret will be 1. 
   That means that print can be used as part of a more complex expression where echo cannot. 
   An example from the PHP Manual: $b ? print "true" : print "false";
   
   print is also part of the precedence table which it needs to be if it is to be used within a complex expression. 
   It is just about at the bottom of the precedence list though. Only "," AND, OR and XOR are lower.
	
   Parameter(s) - The grammar is: echo expression [, expression[, expression] ... ] But echo ( expression, expression ) is not valid. 
   This would be valid: echo ("howdy"),("partner"); the same as: echo "howdy","partner"; 
   (Putting the brackets in that simple example serves no purpose since there is no operator precedence issue with a single term like that.)
   
   So, echo without parentheses can take multiple parameters, which get concatenated:
   echo  "and a ", 1, 2, 3;   // comma-separated without parentheses
   echo ("and a 123");        // just one parameter with parentheses
   
   print() can only take one parameter:
   print ("and a 123");
   print  "and a 123";

Here is the detail of all the arguments:

Q. what is output buffering  and related topics
A. Output buffering means PHP does not send output (echo/print) directly to the browser.
	Instead, it stores the output in memory (buffer) and sends it later.
	Why Use Output Buffering?
	1. Prevent "Headers already sent" error
		ob_start();
		echo "Hello";
		header("Location: test.php"); // Works fine
		ob_end_flush();
	2. Modify Output Before Sending
		ob_start();
		echo "Hello World";
		$content = ob_get_contents();
		ob_end_clean();
		echo strtoupper($content); // HELLO WORLD
	3. Capture Output as Variable
		ob_start();
		echo "This is captured";
		$data = ob_get_clean();
		echo $data;
		
	Important functions : 
		ob_start()	Start buffering
		ob_get_contents()	Get buffer content
		ob_get_clean()	Get + clean buffer
		ob_end_flush()	Send output and end buffer
		ob_end_clean()	Discard buffer
		ob_flush()	Flush buffer

	Output Buffer Flow
	ob_start();      // Start
	echo "Data";     // Stored in buffer
	$content = ob_get_contents(); // Get buffer
	ob_end_clean(); // Clear buffer
	
Buffer Levels (Advanced)
ob_start();
ob_start();
echo ob_get_level(); // 2

Q. What are the common security vulnerabilities in PHP applications? How do you mitigate them?
A. Modern PHP is secure by design, but most vulnerabilities come from unsafe coding patterns
	1) SQL Injection
	2) Cross-Site Scripting (XSS)  use - echo htmlspecialchars($_GET['name'], ENT_QUOTES, 'UTF-8');
	3) Cross-Site Request Forgery (CSRF)
		// Generate
		$_SESSION['csrf'] = bin2hex(random_bytes(32));

		// Validate
		if ($_POST['csrf'] !== $_SESSION['csrf']) {
			die('Invalid CSRF token');
		}
	4) File Inclusion (LFI/RFI)
		Problem: Including files based on user input.
	5) File Upload Vulnerabilities
		Problem: Uploading executable/malicious files.
	6) Session Hijacking / Fixation
		Problem: Stolen or fixed session IDs.
	7) Weak Password Storage
		Problem: Using MD5/SHA1.
	8) Command Injection
		Problem: Passing user input to shell.
	9) Insecure Deserialization
		$obj = unserialize($_POST['data']); // NO 
		
		Fix:
		Avoid unserialize on user input
		Use JSON instead (json_decode)
		If needed, use allowed classes
	10) Information Disclosure
		Problem: Showing errors/stack traces to users.
	11) Improper Access Control
	12) Clickjacking
	
	Security Best Practices (Checklist)
	✅ Validate input (type, length, whitelist)
	✅ Escape output (HTML, JS, URL contexts)
	✅ Use prepared statements
	✅ Use HTTPS everywhere
	✅ Harden sessions (secure, httponly, samesite)
	✅ Use password_hash()
	✅ Limit file uploads (type, size, location)
	✅ Use security headers (CSP, HSTS, X-Frame-Options)
	✅ Keep PHP & dependencies updated (Composer)
	✅ Least privilege (DB, filesystem)
	✅ Log errors, don’t display

Q. Explain how you would implement secure password storage in PHP. Discuss hashing algorithms and their best practices.
A. Storing passwords securely means never saving plain text, but instead storing a strong, salted hash using modern algorithms.
	User enters password → Hash it → Save hash
	Login → Verify password against hash
	
	Recommended Way in PHP : Use built-in functions:
	password_hash()
	password_verify()

	$password = "mySecurePassword";
	$hash = password_hash($password, PASSWORD_DEFAULT);
	echo $hash;
	
	$input = "mySecurePassword";
	if (password_verify($input, $hash)) {
		echo "Login successful";
	} else {
		echo "Invalid password";
	}
	
	Hashing Algorithms : 
	1. Bcrypt (Default) : password_hash($password, PASSWORD_BCRYPT);
	2. Argon2 (Best Modern Option 🚀) : password_hash($password, PASSWORD_ARGON2ID);
	
	Algorithm	Security		Speed		Recommended
	MD5			❌ Weak			Fast		❌ Never use
	SHA1		❌ Weak			Fast		❌ Avoid
	Bcrypt		✅ Strong		Medium		✅ Yes
	Argon2		🔥 Strongest		Slower		✅ Best

	Best Practices
	1. Never Store Plain Passwords
	2. Never Use MD5/SHA1
	3. Use password_hash()
	4. Use password_verify()
	5. Rehash When Needed	: 
		if (password_needs_rehash($hash, PASSWORD_DEFAULT)) {
			$hash = password_hash($password, PASSWORD_DEFAULT);
		}
	6. Use Strong Password Policy : Minimum length (8–12+), Mix of characters, Avoid common passwords
	7. Use HTTPS	: Prevent password sniffing
	8. Add Rate Limiting : Prevent brute force attacks
